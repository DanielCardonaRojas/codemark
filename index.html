Codemark

A structural bookmarking system for code.

Get Started View Source

Why Codemark?

Line-number bookmarks break on any edit. Codemark bookmarks self-heal because they store tree-sitter queries that identify code by its AST shape.

Durable References

Identify code structurally (AST queries), not positionally (file:line). A renamed function is still found even if it moved across the file.

Self-Healing

Layered resolution (exact match → relaxed query → content hash fallback) means bookmarks degrade gracefully rather than breaking silently.

Unix Philosophy

Produces line-oriented, pipe-friendly JSON output. Integrates naturally with fzf, television, and custom shell scripts.

Semantic Search

Natural language queries like "where's the login logic?" return relevant bookmarks using local vector embeddings in SQLite.

Installation

Codemark is a self-contained binary written in Rust. Requires Rust 1.75+.

Homebrew (macOS/Linux)

brew tap DanielCardonaRojas/codemark
brew install codemark

Cargo

cargo install --path .

Quick Start

Easily bookmark and resolve code references from your terminal.

# Bookmark a function by line number
codemark add --file src/auth.rs --range 42 --tag auth --note "Token validation entry point"

# Resolve — find where the code is now
codemark resolve a1b2

# Search by meaning (semantic search)
codemark search --semantic "database connection initialization"

# Preview with bat (syntax highlighted)
codemark preview a1b2 | jq -r '"bat --highlight-line \(.data.line_range) \(.data.file_path)"' | sh

Supported Languages

Codemark uses Tree-sitter to parse source files. Language is auto-detected from file extension.

  • Rust (.rs)
  • Swift (.swift)
  • TypeScript (.ts, .tsx)
  • Python (.py)
  • Go (.go)
  • Java (.java)
  • C# (.cs)
  • Dart (.dart)

How it works

  1. Parse the file with tree-sitter to build an Abstract Syntax Tree (AST).
  2. Find the target node covering the provided line or byte range.
  3. Generate a unique S-expression query for that node.
  4. Store the query, content hash, and semantic metadata in a local SQLite database.
  5. Resolve later by running the query against the current file state.
